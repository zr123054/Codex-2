# PPTX 文件在分支里显示“不支持二进制”说明与解决方案

## 为什么会出现这个提示
`*.pptx` 本质上是二进制打包文件（OpenXML ZIP），很多代码评审页面无法像文本一样逐行展示 diff，
所以会显示“Binary file not shown / 不支持二进制文件预览”。

## 本仓库的处理方式
- **不再把生成后的 `.pptx` 二进制文件直接提交到 Git**。
- 保留 `generate_ppt.py`，需要时本地一键生成可用的 PPT。

## 如何生成可直接使用的 PPT
在仓库根目录执行：

```bash
python generate_ppt.py
```

生成文件：
- `2026上海国际路亚展-汇报模板.pptx`

## 这样做的好处
- PR diff 可读（仅展示脚本和文本改动）。
- 避免二进制文件导致仓库体积膨胀。
- 模板结构可追踪、可审阅、可复现。

## 如果你现在“仍然”看到不支持二进制
这通常不是文件内容问题，而是**远端分支历史还没被替换**。请执行：

```bash
git push --force-with-lease origin <你的分支名>
```

然后在 PR 页面点击 **Update branch / 重新加载**。

> 关键点：必须 force push 让远端分支丢弃曾经包含 `.pptx` 的旧提交；
> 仅仅在本地删除文件、普通 push 不一定能清掉历史中的二进制变更提示。
